# Requirement Defect Taxonomy Table

| Requirement                        | Defect Type           | Description                                     |
|------------------------------------|-----------------------|-------------------------------------------------|
| "System must be secure"            | Non-verifiable        | No measurable criteria defined                  |
| "System should be fast"            | Ambiguity             | No performance threshold specified              |
| "Votes must be anonymous"          | Incomplete            | No mechanism defined                            |
| "User authentication required"     | Inconsistency         | Does not specify method (password, token, etc.) |
| "System prevents duplicate voting" | Partial specification | Does not define how prevention is enforced      |
